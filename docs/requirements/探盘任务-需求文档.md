## 1. 业务背景

### 1.1 探盘任务目的

| 目的 | 说明 |
|-----|------|
| **内容生产** | 让经纪人在探盘时同步生成新媒体获客内容 |
| **产品熟悉** | 让经纪人真实了解楼盘，方便为客户介绍产品、解答问题 |

### 1.2 内容产出类型

| 渠道 | 内容类型 |
|-----|---------|
| 小红书 | 文案、封面图 |
| 公众号 | 文案，配图 |
| 短视频 | 脚本、视频 |
| 朋友圈 | 文案，配图/视频 |

## 2. 用户角色

| 角色 | 说明 | 端 |
|-----|------|-----|
| 新房维护人员 | 上线新房项目 | PC端 |

| 经纪人/店长 | 执行探盘任务 | 小程序端 |

| 端 | 功能范围 |
|----|---------|
| **PC管理后台** | 任务创建、批量下发、批量导出、批量修改人员、审核管理 |

## 4. 核心业务流程

### 4.1 任务自动生成规则

- **触发时机**：新盘项目审核通过、自动上线后，系统自动生成待分发探盘任务
- **生成内容**：任务默认填充项目基础信息（楼盘名称、地址、均价、户型等）
- **后续流程**：运营人员收到通知 → 编辑任务信息 → 指定执行/通知人员 → 下发

### 4.2 任务下发流程

```
1. 运营人员在PC端创建/编辑探盘任务
2. 选择楼盘（自动带入楼盘信息）
3. 配置人员（执行人员+通知人员）
4. 配置打卡规则（范围/点位/完成规则）
5. 配置结果流向（文案/脚本/封面/视频）
6. 点击「下发」
7. 执行人员收到小程序通知
8. 经纪人到现场执行任务
```

### 4.3 任务执行流程（小程序侧）

| 阶段 | 名称 | 触发条件 | 具体内容 |
|------|------|----------|----------|
| 阶段1 | 定位打卡 | 点击「开始执行」 | 系统校验当前位置是否在楼盘定位范围内 |
| 阶段2 | 点位打卡 | 阶段1完成后进入 | 按配置的打卡点位依次完成（拍照/视频/录音/文本） |
| 阶段3 | 文案确认 | 所有必填点位完成后进入 | AI生成文案初稿，用户确认或修改 |
| 阶段4 | 成果生成 | 阶段3确认后自动进入 | 根据结果流向配置自动生成内容 |
| 提交 | 提交审核 | 阶段4完成后点击「提交」 | 状态变为「已提交」 |

## 5. 探盘任务状态

### 5.1 核心状态（7个）

| 状态 | 说明 | PC端可见操作 |
|------|------|-------------|
| 待分发 | 创建后未下发 | 下发、编辑、删除 |
| 待执行 | 已下发等待执行 | 查看详情、催办、取消 |
| 进行中 | 经纪人在执行中 | 查看进度、催办 |
| 已提交 | 经纪人已提交，等待审核 | 查看结果、审核 |
| 已完成 | 审核通过 | 查看结果 |
| 已驳回 | 审核不通过，需重新修改 | 重新下发 |
| 已逾期 | 超时未完成 | 查看进度、催办 |

### 5.2 状态流转图

```
待分发 → 待执行 → 进行中 → 已提交 → 已完成/已驳回
                          ↘ 已逾期（可继续执行）
                          ↘ 已驳回（可重新下发→待执行）
```

### 5.3 打卡规则

- 必须先完成阶段1「定位打卡」，才能进入阶段2「点位打卡」
- 「必须打卡」的点位不可跳过；「可选打卡」的点位可跳过
- 需在「点位打卡完成时间」内完成所有点位打卡

### 5.4 完成规则

| 规则 | 说明 |
|------|------|
| 全部点位必须完成 | 所有配置的必填点位都完成才能进入阶段3 |
| 部分点位完成即可 | 完成配置的必填点位的50%即可进入阶段3 |

## 6. 打卡配置

### 6.1 PC端打卡模版配置

| 配置项 | 说明 | 默认值 |
|-------|------|-------|
| 打卡范围 | 定位打卡范围半径（米） | 500米 |
| 打卡规则 | 先定位打卡→再点位打卡（必须完成定位打卡才能开始点位打卡） | - |
| 点位打卡完成时间 | 所有点位打卡完成的时间限制（相对阶段1开始的时长） | - |
| 完成规则 | 全部点位必须完成 / 部分点位完成即可 | - |
| 结果流向配置 | 多平台文案创作、视频脚本生成、封面内容生成、视频生成（可多选） | - |

### 6.2 打卡点位配置

| 配置项 | 说明 |
|-------|------|
| 点位列表 | 当前任务的打卡点位，支持增删改操作 |
| 开启状态 | 该点位是否必须打卡 |
| 打卡内容 | 该点位需采集的内容类型：照片/视频/录音/文本 |
| 备注说明 | 该点位的特殊要求或说明 |

**打卡内容类型**：
- 拍照：需配置最少/最多张数
- 视频：需配置最短/最长时长
- 录音：需配置最短/最长时长
- 文本：输入文字说明

## 7. 结果流向配置

| 配置项 | 阶段4生成内容 |
|--------|--------------|
| 多平台文案创作 | 适用于不同平台（抖音、小红书、微信公众号等）的探盘文案 |
| 视频脚本生成 | 探盘视频的完整脚本，包含开场白、卖点讲解、结尾引导 |
| 封面内容生成 | 探盘报告的封面图设计 |
| 视频生成 | 自动合成的探盘讲盘视频（AI配音+素材合成） |

## 8. 审核流程

### 8.1 审核时机

- 执行人员在微信小程序完成所有阶段执行后，点击「提交」，任务状态变为「已提交」
- 只有「已提交」状态的任务，审核人员才能进行审核操作
- 审核通过 → 状态变为「已完成」
- 审核驳回 → 状态变为「已驳回」，执行人员修改后可重新提交

### 8.2 审核角色

| 角色 | 审核权限 | 说明 |
|------|----------|------|
| 区域经理 | 本区域任务审核 | 一级审核，可通过/驳回 |
| 总部运营 | 全量内容巡查 | 二级审核，抽查+最终发布确认 |
| AI内容质检 | 自动审核 | 敏感词/违规内容自动检测（系统层面） |

### 8.3 审核标准

**定量标准（可量化）：**
- 文案字数：小红书 ≥300字，朋友圈 ≥50字
- 图片：封面图分辨率 ≥1080×1080
- 必填字段：楼盘名称/均价/地址/联系方式

**定性标准（主观判断）：**
- 内容是否与楼盘实际情况一致
- 文案是否具有获客吸引力
- 是否存在违规宣传语（限时/最低价等）

### 8.4 驳回与重新下发

- 驳回必须填写原因（系统提供常用原因选项，也支持自定义）
- 驳回后自动通知经纪人，可直接在小程序修改后重新提交
- 重新下发时默认填充原执行人员（可修改）

## 9. 数据统计指标

### 9.1 基础指标

| 指标 | 计算公式 | 说明 |
|-----|---------|------|
| 总任务数 | SUM | 所有创建的任务数 |
| 待分发 | COUNT(status=待分发) | 还未下发的任务 |
| 待执行 | COUNT(status=待执行) | 已下发等待执行 |
| 进行中 | COUNT(status=进行中) | 正在执行的任务 |
| 已提交 | COUNT(status=已提交) | 已提交待审核 |
| 已完成 | COUNT(status=已完成) | 审核通过的任务 |
| 已驳回 | COUNT(status=已驳回) | 被驳回的任务 |
| 已逾期 | COUNT(status=已逾期) | 超时的任务 |
| 完成率 | 已完成/总任务 × 100% | - |
| 超时率 | 已逾期/总任务 × 100% | - |
| 通过率 | 审核通过/已提交 × 100% | - |

### 9.2 扩展指标

| 指标类别 | 指标名称 | 说明 |
|----------|----------|------|
| 任务执行 | 平均完成时长 | 从下发到提交的时长 |
| 内容质量 | 一次通过率 | 无需返修即通过的比例 |
| 内容产出 | 人均产出量 | 人均完成探盘任务数 |
| 内容产出 | 内容发布率 | 审核通过后实际发布的比例 |
| 平台效果 | 各平台发布数 | 小红书/公众号/朋友圈分别统计 |
| 平台效果 | 阅读量/点赞量 | 从平台回传（需对接平台API或手动录入） |

### 9.3 分析维度

| 维度 | 说明 |
|-----|------|
| 区域维度 | 各区域的探盘任务完成情况 |
| 经纪人维度 | 各经纪人的任务完成情况 |
| 楼盘维度 | 各楼盘的探盘任务情况 |

## 10. 超时机制

| 超时时间 | 触发动作 |
|----------|----------|
| 下发后24小时未开始 | 发送提醒通知（小程序+短信） |
| 下发后48小时未完成 | 区域经理收到催报通知 |
| 超时7天未完成 | 任务自动降级（沉默任务，不计入完成率） |
| 超时30天未完成 | 任务自动关闭，标记「未执行」 |

**超时不影响绩效**：超时任务不扣分，仅做数据跟踪。

## 16. 产品建议（待实施）

1. **探盘任务模板化**：同一楼盘可复制历史任务模板
2. **批量下发**：支持按区域/门店/经纪人批量下发
3. **AI内容预检**：提交前自动检测敏感词/合规问题
4. **内容复用**：同一楼盘多次探盘的内容可沉淀，形成楼盘内容库
5. **跨平台发布助手**：审核通过后支持一键发布至各平台
6. **探盘距离校验**：GPS定位与楼盘坐标偏差超过500米时提示
