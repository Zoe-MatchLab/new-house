## 1. 业务背景

### 1.1 探盘任务目的

| 目的 | 说明 |
|-----|------|
| **内容生产** | 让经纪人在探盘时同步生成新媒体获客内容 |
| **产品熟悉** | 让经纪人真实了解楼盘，方便为客户介绍产品、解答问题 |

### 1.2 内容产出类型

| 渠道 | 内容类型 |
|-----|---------|
| 小红书 | 文案、配图 |
| 公众号 | 文案、配图 |
| 朋友圈 | 文案、配图 |
| 短视频 | 脚本、视频 |

---

## 2. 用户角色

| 角色 | 说明 | 端 |
|-----|------|-----|
| 新房维护人员 | 上线新房项目 | PC端 |
| 运营人员 | 创建/下发探盘任务 | PC端 |
| 区域经理 | 组织经纪人探盘、催办 | PC端 + 小程序端 |
| 经纪人/店长 | 执行探盘任务 | 小程序端 |
| 公司总部 | 查看数据 | PC端 |

---

## 3. 端侧分工

| 端 | 功能范围 |
|----|---------|
| **PC管理后台** | 任务创建、批量下发、批量导出、批量修改人员、暂停任务 |
| **小程序侧** | 单条任务配置与下发、任务执行（定位打卡+点位校验）、内容生成 |

---

## 4. 任务类型

小程序端展示两类任务：

| 任务类型 | 说明 | 状态 |
|---------|------|------|
| **下发任务** | 方便管理层移动端作业 | 待下发、进行中、已完成、已失效 |
| **探盘任务** | 用于经纪人去执行 | 未开始、进行中、已完成、已失效、即将到期、催办 |

---

## 5. 探盘任务状态

### 5.1 下发任务状态（PC端）

| 状态 | 说明 | PC端可见操作 |
|------|------|-------------|
| 待下发 | 创建后未下发 | 下发、编辑、删除 |
| 进行中 | 经纪人在执行中 | 查看进度、催办、暂停 |
| 已完成 | 任务已完成 | 查看结果 |
| 已失效 | 任务已过期/暂停 | 查看记录 |

### 5.2 探盘任务状态（小程序端）

| 状态 | 说明 | 小程序端显示 |
|------|------|-------------|
| 未开始 | 任务未开始执行 | 显示为"未开始" |
| 进行中 | 经纪人在执行中 | 显示进度 |
| 即将到期 | 任务即将到期 | 显示提醒 |
| 催办 | 被催办 | 显示催办标记 |
| 已完成 | 任务已完成 | 显示完成状态 |
| 已失效 | 任务已过期/暂停 | 显示失效状态 |

### 5.3 状态流转图

```
【下发任务】
待下发 → 进行中 → 已完成
              ↓
           已失效（超时/暂停）

【探盘任务】
未开始 → 进行中 → 已完成
              ↓
           已失效（超时）
```

## 6. 核心业务流程

### 6.1 PC端流程

```
创建任务 → 下发（下发后还可催办） → 暂停任务（暂停后不可重启，删除需谨慎） → 查看任务
```

**PC端功能说明**：
- 创建任务：选择楼盘、配置人员、配置打卡范围
- 下发：批量/单条下发，下发后给经纪人发通知
- 催办：进行中的任务可催办，超时后不可催办
- 暂停：暂停后任务不可重启，删除需谨慎
- 查看：查看任务进度、数据统计

### 6.2 小程序端流程

**探盘任务包含3个子任务**：

| 子任务 | 说明 |
|-------|------|
| 探盘路书 | 主要探盘流程 |
| 话术训练 | 话术练习 |
| 邀约录音分析 | 录音分析 |

**探盘路书执行流程**：

```
进入探盘路书任务
      ↓
收集点位资料（每次上传时校验是否在指定点位范围内）
      ↓
提交资料
      ↓
选择需要生成的内容（多渠道）
      ↓
生成（调用Coze工作流API）
      ↓
确认内容（算完成探盘路书的任务）
      ↓
生成视频/配图
```

## 7. 打卡配置

### 7.1 PC端配置

| 配置项 | 说明 | 默认值 |
|-------|------|-------|
| 打卡范围 | 定位打卡范围半径（米） | 500米 |
| 点位配置 | 配置点位位置和范围 | - |

### 7.2 小程序端执行

| 功能 | 说明 |
|-----|------|
| 定位打卡 | 到现场后校验是否在楼盘定位范围内 |
| 点位校验 | 每次点位资料上传时，校验是否在指定点位范围内；不在范围内则提醒用户需要在指定范围内上传 |

## 8. 内容生成

### 8.1 生成方式

**触发**：经纪人手动填写探盘信息 → 提交 → 触发Coze工作流API

### 8.2 生成内容

| 内容类型 | 说明 | 生成方式 |
|---------|------|---------|
| 小红书文案 | 适用于小红书平台的探盘种草文案 | Coze工作流 |
| 公众号文案 | 适用于微信公众号的探盘介绍文案 | Coze工作流 |
| 朋友圈文案 | 适用于朋友圈的简短推广文案 | Coze工作流 |
| 短视频脚本 | 探盘视频的完整脚本 | Coze工作流 |
| 小红书配图 | 小红书封面图 | Coze工作流 |
| 公众号配图 | 公众号封面图 | Coze工作流 |
| 朋友圈配图 | 朋友圈封面图 | Coze工作流 |
| 视频 | 探盘视频 | Coze工作流 |

### 8.3 Coze工作流输入

- 楼盘基本信息（名称、地址、均价、户型）
- 经纪人填写的探盘信息（配套、卖点、现场情况）
- 现场照片/视频素材

## 9. 审核流程

**说明**：下发任务不设计审核流程

- 小程序端：无审核动作
- PC端：无审核功能

---

## 10. 超时机制

### 10.1 小程序端

| 提醒类型 | 说明 |
|---------|------|
| 到期提醒 | 由下发任务时配置，最多3个提醒时间（轻度、中度、严重） |
| 任务过期提醒 | 任务过期表示任务失效 |

### 10.2 PC端

| 规则 | 说明 |
|------|------|
| 超时后不可催办 | 任务超时后，催办按钮不可用 |
| 只对进行中任务催办 | 只有状态为"进行中"的任务可以催办 |

---

## 11. 通知机制

### 11.1 小程序端通知

| 通知类型 | 说明 |
|---------|------|
| 任务通知 | 收到新任务、被催办等 |
| 提醒通知 | 到期提醒（轻度/中度/严重） |

### 11.2 PC端通知

| 通知类型 | 说明 |
|---------|------|
| 下发通知 | 下发任务时给经纪人发通知 |
| 催办通知 | 催办时给经纪人发通知 |
| 暂停通知 | 暂停任务时给经纪人发通知 |

---

## 12. 数据统计指标

### 12.1 基础指标

| 指标 | 计算公式 | 说明 |
|-----|---------|------|
| 总任务数 | SUM | 所有创建的任务数 |
| 待下发 | COUNT(status=待下发) | 还未下发的任务 |
| 进行中 | COUNT(status=进行中) | 正在执行的任务 |
| 已完成 | COUNT(status=已完成) | 已完成的任务 |
| 已失效 | COUNT(status=已失效) | 已过期/暂停的任务 |
| 完成率 | 已完成/总任务 × 100% | - |

### 12.2 分析维度

| 维度 | 说明 |
|-----|------|
| 区域维度 | 各区域的探盘任务完成情况 |
| 经纪人维度 | 各经纪人的任务完成情况 |
| 楼盘维度 | 各楼盘的探盘任务情况 |
