## 1. 业务背景

### 1.1 探盘任务目的

| 目的 | 说明 |
|-----|------|
| **内容生产** | 让经纪人在探盘时同步生成新媒体获客内容 |
| **产品熟悉** | 让经纪人真实了解楼盘，方便为客户介绍产品、解答问题 |

### 1.2 内容产出类型

| 渠道 | 内容类型 |
|-----|---------|
| 小红书 | 文案、封面图 |
| 公众号 | 文案，配图 |
| 朋友圈 | 文案，配图/视频 |

## 2. 用户角色

| 角色 | 说明 | 端 |
|-----|------|-----|
| 新房维护人员 | 上线新房项目 | PC端 |

| 端 | 功能范围 |
|----|---------|
| **PC管理后台** | 任务创建、批量下发、批量导出、批量修改人员、审核管理 |
| **小程序侧** | 单条任务配置与下发、任务执行、提交结果 |

## 4. 核心业务流程

### 4.1 任务自动生成规则

- **触发时机**：新盘项目审核通过、自动上线后，系统自动生成待下发探盘任务
- **生成内容**：任务默认填充项目基础信息（楼盘名称、地址、均价、户型等）
- **后续流程**：运营人员收到通知 → 编辑任务信息 → 指定执行/通知人员 → 下发

### 4.2 任务下发流程

```
1. 运营人员在PC端创建/编辑探盘任务
2. 选择楼盘（自动带入楼盘信息）
3. 配置人员（执行人员+通知人员）
4. 配置打卡规则（范围/点位）
5. 点击「下发」
6. 执行人员收到小程序通知
7. 经纪人到现场执行任务
```

### 4.3 任务执行流程（小程序侧）

| 步骤 | 名称 | 触发条件 | 具体内容 |
|------|------|----------|----------|
| 步骤1 | 定位打卡 | 点击「开始执行」 | 系统校验当前位置是否在楼盘定位范围内 |
| 步骤2 | 填写信息 | 手动填写 | 楼盘基本信息、配套信息、卖点等 |
| 步骤3 | 提交 | 点击「提交」 | 提交后调用Coze工作流生成内容 |
| 步骤4 | 审核 | 审核通过 | 内容发布 |

**内容生成说明**：
- 经纪人在小程序侧手动填写探盘信息（楼盘信息、配套、卖点等）
- 提交后系统调用**Coze工作流API**生成新媒体内容
- 生成的文案/封面等内容进入内容库，由运营审核后发布

## 5. 探盘任务状态

### 5.1 核心状态（4个）

| 状态 | 说明 | PC端可见操作 |
|------|------|-------------|
| 待下发 | 创建后未下发 | 下发、编辑、删除 |
| 进行中 | 经纪人在执行中 | 查看进度、催办 |
| 已完成 | 任务已完成 | 查看结果 |
| 已失效 | 任务已过期/取消 | 查看记录 |

### 5.2 状态流转图

```
待下发 → 进行中 → 已完成
                ↓
              已失效
```

## 6. 打卡配置

### 6.1 PC端打卡模版配置

| 配置项 | 说明 | 默认值 |
|-------|------|-------|
| 打卡范围 | 定位打卡范围半径（米） | 500米 |
| 打卡规则 | 必须先完成定位打卡才能开始点位打卡 | - |
| 点位列表 | 当前任务的打卡点位 | - |

### 6.2 打卡点位配置

| 配置项 | 说明 |
|-------|------|
| 点位列表 | 当前任务的打卡点位，支持增删改操作 |
| 开启状态 | 该点位是否必须打卡 |
| 打卡内容 | 该点位需采集的内容类型：照片/视频/录音/文本 |

## 7. 内容生成

**生成方式**：经纪人手动填写探盘信息后，调用Coze工作流API自动生成

**生成内容**：

| 内容类型 | 说明 | 生成方式 |
|---------|------|---------|
| 小红书文案 | 适用于小红书平台的探盘种草文案 | Coze工作流 |
| 公众号文案 | 适用于微信公众号的探盘介绍文案 | Coze工作流 |
| 朋友圈文案 | 适用于朋友圈的简短推广文案 | Coze工作流 |
| 封面图 | 探盘内容的封面图 | Coze工作流 |

**Coze工作流输入**：
- 楼盘基本信息（名称、地址、均价、户型）
- 经纪人填写的探盘信息（配套、卖点、现场情况）
- 现场照片/视频素材

**Coze工作流输出**：
- 各平台适配的文案
- 封面图

## 8. 审核流程

### 8.1 审核角色

| 角色 | 审核权限 | 说明 |
|------|----------|------|
| 区域经理 | 本区域任务审核 | 一级审核，可通过/驳回 |
| 总部运营 | 全量内容巡查 | 二级审核，抽查+最终发布确认 |

### 8.2 审核标准

**定量标准（可量化）：**
- 文案字数：小红书 ≥300字，朋友圈 ≥50字
- 图片：封面图分辨率 ≥1080×1080
- 必填字段：楼盘名称/均价/地址/联系方式

**定性标准（主观判断）：**
- 内容是否与楼盘实际情况一致
- 文案是否具有获客吸引力
- 是否存在违规宣传语（限时/最低价等）

## 9. 数据统计指标

### 9.1 基础指标

| 指标 | 计算公式 | 说明 |
|-----|---------|------|
| 总任务数 | SUM | 所有创建的任务数 |
| 待下发 | COUNT(status=待下发) | 还未下发的任务 |
| 进行中 | COUNT(status=进行中) | 正在执行的任务 |
| 已完成 | COUNT(status=已完成) | 已完成的任务 |
| 已失效 | COUNT(status=已失效) | 已过期/取消的任务 |
| 完成率 | 已完成/总任务 × 100% | - |

### 9.2 分析维度

| 维度 | 说明 |
|-----|------|
| 区域维度 | 各区域的探盘任务完成情况 |
| 经纪人维度 | 各经纪人的任务完成情况 |
| 楼盘维度 | 各楼盘的探盘任务情况 |

## 10. 超时机制

| 超时时间 | 触发动作 |
|----------|----------|
| 下发后24小时未开始 | 发送提醒通知 |
| 下发后48小时未完成 | 区域经理收到催报通知 |
| 超时7天未完成 | 任务自动失效 |
