## 1. 业务背景

### 1.1 探盘任务目的

| 目的 | 说明 |
|-----|------|
| **内容生产** | 让经纪人在探盘时同步生成新媒体获客内容 |
| **产品熟悉** | 让经纪人真实了解楼盘，方便为客户介绍产品、解答问题 |

### 1.2 内容产出类型

| 渠道 | 内容类型 |
|-----|---------|
| 小红书 | 文案、封面图 |
| 公众号 | 文案、配图 |
| 短视频 | 脚本、视频 |
| 朋友圈 | 文案、配图/视频 |

## 2. 用户角色

| 角色 | 说明 | 端 |
|-----|------|-----|
| 新房维护人员 | 上线新房项目，下发探盘任务 | PC端 |
| 区经（区域经理） | 组织经纪人探盘 | PC端 + 小程序端 |
| 经纪人/店长 | 执行探盘任务 | 小程序端 |
| 公司总部 | 查看数据、审核内容 | PC端 |

---

## 3. 核心业务流程

### 3.1 任务下发流程

```
1. 新房维护人员 在PC端「新房管理」上线新盘
2. 上线时选择「可下发探盘任务的人员」（从组织架构选择）
3. 区经在PC端/小程序端 创建探盘任务
4. 系统通知被选中的经纪人/店长
5. 经纪人到现场执行探盘任务
```

### 3.2 任务执行流程

```
1. 经纪人到达楼盘
2. 小程序自动获取定位（隐性，不展示给用户）
3. 校验定位是否在指定范围内
   - 不在范围内 → 提示「请到现场执行任务」
4. 拍照上传现场照片
5. 填写楼盘信息（手动/AI辅助）
6. AI根据探盘信息生成新媒体内容
7. 提交任务
```

## 4. 探盘任务状态

| 状态 | 说明 | 状态码 |
|-----|------|-------|
| 待下发 | 任务创建后，未下发 | pending |
| 待执行 | 已下发，等待经纪人执行 | waiting |
| 进行中 | 经纪人在执行中 | in_progress |
| 已完成 | 任务已完成提交 | completed |
| 已超时 | 任务超时未完成 | timeout |
| 已取消 | 任务被取消 | cancelled |

### 4.1 状态流转图

```
待下发 → 待执行 → 进行中 → 已完成
                ↓           ↓
              已超时     已取消
```

## 8. 探盘任务状态（PM Agent建议）


### 8.1 核心状态（6个）

| 状态 | 含义 | 触发动作 |
|------|------|----------|
| 待下发 | 任务已创建，等待排期下发 | 管理员/区域经理创建任务 |
| 已下发 | 经纪人已收到任务，待执行 | 系统推送至小程序 |
| 进行中 | 经纪人已开始执行（进入阶段1打卡） | 经纪人点击"开始探盘" |
| 已完成 | 经纪人提交成果（4阶段全部完成） | 第4阶段成果提交 |
| 已审核 | 管理员/区域经理审核通过 | 审核人点击"通过" |
| 已驳回 | 审核不通过，需重新修改 | 审核人点击"驳回"并填写原因 |

### 8.2 子阶段状态

| 阶段 | 子状态 | 流转 |
|------|--------|------|
| 阶段1 打卡 | 待打卡→定位中→已打卡→打卡异常 | 拍照+定位 |
| 阶段2 素材 | 待上传→上传中→已上传 | 拍照/相册 |
| 阶段3 文案 | 待撰写→撰写中→待确认→已确认 | AI生成→用户确认 |
| 阶段4 封面 | 待生成→生成中→已生成 | AI生成封面图 |

### 8.3 超时状态
- **超时未开始**：任务已下发超过X小时但未进入"进行中"
- **超时未完成**：进行中状态超过规定时长，自动提醒+降级标记

## 9. 内容生成流程（PM Agent建议）

### 9.1 触发流程
```
任务下发 → 经纪人执行 → 阶段1打卡(定位+拍照)
         → 阶段2上传素材(多图)
         → 阶段3 AI生成文案(自动) → 用户确认/修改
         → 阶段4 AI生成封面(自动) → 用户确认/修改
         → 提交成果 → 审核 → 多平台发布
```

### 9.2 内容生成方式

| 内容类型 | 触发方式 | 说明 |
|----------|----------|------|
| 小红书文案 | **自动生成** | 探盘完成后AI立即生成，支持修改 |
| 公众号文案 | **自动生成** | 与小红书共用素材池，可单独生成 |
| 短视频脚本 | **手动触发** | 用户主动选择"生成视频脚本" |
| 朋友圈文案 | **自动生成** | 基于探盘数据快速生成短文案 |
| 封面图 | **自动生成** | AI根据楼盘信息+素材生成 |

### 9.3 AI能力来源
- **文案生成**：调用 AI 大模型（GPT/Claude/国产模型），输入楼盘基础信息+经纪人探盘素材
- **封面生成**：调用 AI 图像生成能力，输入楼盘名称+调性标签
- **多平台适配**：文案生成时按平台使用不同 Prompt 模板
- **调用时机**：阶段2素材上传完成后，后台立即并行调用

## 11. 审核流程（PM Agent建议）

### 11.1 是否需要审核
**建议：需要审核**，理由：
- AI生成内容可能存在信息错误/楼盘数据偏差
- 涉及品牌形象（朋友圈/公众号内容代表公司调性）
- 小红书/公众号属于公开发布，内容风险需把控

### 11.2 审核角色

| 角色 | 审核权限 | 说明 |
|------|----------|------|
| 区域经理 | 本区域任务审核 | 一级审核，可通过/驳回 |
| 总部运营 | 全量内容巡查 | 二级审核，抽查+最终发布确认 |
| AI内容质检 | 自动审核 | 敏感词/违规内容自动检测（系统层面） |

### 11.3 审核标准

**定量标准（可量化）：**
- 文案字数：小红书 ≥300字，朋友圈 ≥50字
- 图片：封面图分辨率 ≥1080×1080
- 必填字段：楼盘名称/均价/地址/联系方式

**定性标准（主观判断）：**
- 内容是否与楼盘实际情况一致
- 文案是否具有获客吸引力
- 是否存在违规宣传语（限时/最低价等）

## 12. 其他建议（PM Agent）

### 12.1 探盘频次建议

| 楼盘类型 | 建议频次 | 说明 |
|----------|----------|------|
| 重点新盘 | 每周1次 | 持续更新内容 |
| 普通在售盘 | 每2周1次 | 维持热度 |
| 尾盘/清盘 | 每月1次 | 收尾曝光 |
| 竞品楼盘 | 按需探盘 | 市场监控目的 |

### 12.2 超时机制

```
下发后24小时未开始 → 发送提醒通知
下发后48小时未完成 → 区域经理收到催报通知
超时7天未完成 → 任务自动降级（沉默任务，不计入完成率）
超时30天未完成 → 任务自动关闭，标记"未执行"
```

### 12.3 其他产品建议

1. **探盘任务模板化**：同一楼盘可复制历史任务模板
2. **批量下发**：支持按区域/门店/经纪人批量下发
3. **AI内容预检**：提交前自动检测敏感词/合规问题
4. **内容复用**：同一楼盘多次探盘的内容可沉淀，形成楼盘内容库
5. **跨平台发布助手**：审核通过后支持一键发布至各平台
6. **探盘距离校验**：GPS定位与楼盘坐标偏差超过500米时提示
