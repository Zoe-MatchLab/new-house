## 1. 产品概述

### 1.1 产品背景

随着新房市场竞争加剧，经纪人需要更高效地了解楼盘信息并生成新媒体获客内容。传统探盘模式存在以下问题：

- 经纪人探盘后需要手动整理信息，效率低下
- 内容生产依赖人工撰写，产出质量参差不齐
- 探盘执行过程缺乏管控，难以保证真实性
- 任务下发和执行分散在多个系统，操作繁琐

### 1.2 产品目标

| 目标 | 说明 | 衡量指标 |
|-----|------|---------|
| 提升探盘效率 | 标准化探盘流程，减少人工操作 | 探盘任务完成时长降低30% |
| 保证内容质量 | AI辅助生成高质量新媒体内容 | 内容审核一次通过率≥80% |
| 规范执行过程 | 定位+点位校验，确保探盘真实性 | 定位校验覆盖率100% |
| 统一任务管理 | PC+小程序双端任务管理 | 任务下发到执行闭环率≥95% |

### 1.3 产品定位

探盘任务是一款面向房地产经纪人的探盘管理工具，通过「探盘+内容生成」一体化流程，帮助经纪人高效完成楼盘调研并同步生成新媒体获客内容。

**核心价值**：
- 标准化探盘流程，确保信息采集完整
- AI自动生成内容，降低经纪人文案压力
- 定位+点位校验，保证探盘真实性

## 5. 功能详述

### 5.1 PC端 - 任务管理

#### 5.1.1 任务列表

**路径**：`/pc/task-manage.html`

**功能说明**：
- 展示所有探盘任务，支持多维度筛选
- 显示任务名称、状态、执行人员、截止时间、进度
- 支持批量操作（批量下发、批量导出、批量修改人员）

**筛选条件**：
- 关联楼盘
- 任务状态（待下发、进行中、已完成、已失效）
- 执行人员
- 时间范围

**顶部提示**：
- 待下发任务徽章（橙色）：显示待下发数量，点击自动筛选待下发任务

#### 5.1.2 创建任务

**路径**：`/pc/task-create.html`

**表单字段**：

| 模块 | 字段 | 必填 | 说明 |
|------|------|:----:|------|
| **基础信息** | 任务名称 | ✅ | 如"XX楼盘探盘任务" |
| | 关联楼盘 | ✅ | 从楼盘列表选择 |
| | 优先级 | ❌ | 高/中/低 |
| | 截止时间 | ✅ | 日期时间选择器 |
| **人员配置** | 执行人员 | ✅ | 从组织架构树形选择，支持批量勾选 |
| | 通知人员 | ❌ | 知晓任务进度的知会人员 |
| **打卡配置** | 打卡范围 | ✅ | 定位打卡范围半径（米），默认500米 |
| | 打卡规则 | ✅ | 先定位打卡→再点位打卡 |
| **提醒配置** | 提醒时间 | ❌ | 最多配置3个提醒（轻度、中度、重度） |

#### 5.1.3 任务下发

**下发流程**：
1. 选择执行人员（组织架构树形选择）
2. 确认下发
3. 执行人员收到小程序通知
4. 任务状态变为"进行中"

**批量下发**：
- 勾选多个任务后，可批量选择执行人员下发

**下发后操作**：
- 下发后仍可催办（进行中的任务）
- 下发后不可修改执行人员

#### 5.1.4 任务催办

**规则**：
- 只有"进行中"状态的任务可以催办
- 超时后不可催办
- 催办后发送通知给执行人员

#### 5.1.5 任务暂停

**规则**：
- 暂停后任务状态变为"已失效"
- 暂停后不可重启
- 删除需谨慎操作（二次确认）

### 5.4 小程序端 - 任务执行

#### 5.4.1 探盘路书

**执行流程**：

```
进入探盘路书任务
      ↓
收集点位资料
（每次上传时校验是否在指定点位范围内）
      ↓
提交资料
      ↓
选择需要生成的内容渠道
（小红书/公众号/朋友圈/短视频）
      ↓
AI生成内容
（调用Coze工作流API）
      ↓
确认内容
（确认后算完成探盘路书任务）
      ↓
生成视频/配图
```

#### 5.4.2 定位打卡

**规则**：
- 点击"开始执行"时，系统校验当前位置是否在楼盘定位范围内
- 默认打卡范围：500米
- 不在范围内时，提示"请到现场执行任务"

#### 5.4.3 点位校验

**规则**：
- 每次点位资料上传时，校验是否在指定点位范围内
- 不在范围内时，提醒"需要在指定范围内上传"

#### 5.4.4 子任务

| 子任务 | 说明 |
|-------|------|
| 探盘路书 | 主要探盘流程（点位资料收集+内容生成） |
| 话术训练 | 话术练习 |
| 邀约录音分析 | 录音分析 |

### 5.5 小程序端 - 内容生成

#### 5.5.1 生成方式

**触发时机**：经纪人填写探盘信息并提交后

**调用方式**：调用Coze工作流API

#### 5.5.2 可生成内容

| 内容类型 | 说明 |
|---------|------|
| 小红书文案 | 适用于小红书平台的探盘种草文案 |
| 公众号文案 | 适用于微信公众号的探盘介绍文案 |
| 朋友圈文案 | 适用于朋友圈的简短推广文案 |
| 短视频脚本 | 探盘视频的完整脚本 |
| 小红书配图 | 小红书封面图 |
| 公众号配图 | 公众号封面图 |
| 朋友圈配图 | 朋友圈封面图 |
| 视频 | 探盘视频 |

#### 5.5.3 Coze工作流输入

- 楼盘基本信息（名称、地址、均价、户型）
- 经纪人填写的探盘信息（配套、卖点、现场情况）
- 现场照片/视频素材

#### 5.5.4 内容确认

- 生成后需经纪人确认
- 确认后任务完成

## 8. 数据指标

### 8.1 基础指标

| 指标 | 计算方式 | 说明 |
|-----|---------|------|
| 总任务数 | SUM | 所有创建的任务数 |
| 待下发数 | COUNT(status=待下发) | 还未下发的任务数 |
| 进行中数 | COUNT(status=进行中) | 正在执行的任务数 |
| 已完成数 | COUNT(status=已完成) | 已完成的任务数 |
| 已失效数 | COUNT(status=已失效) | 已过期/暂停的任务数 |
| 完成率 | 已完成数 / 总任务数 × 100% | - |

### 8.2 分析维度

| 维度 | 指标 |
|-----|------|
| 区域维度 | 各区域任务数量、完成率、超时率 |
| 经纪人维度 | 各经纪人任务数量、完成率、平均完成时长 |
| 楼盘维度 | 各楼盘任务数量、完成率、内容产出数量 |
