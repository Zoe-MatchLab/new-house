## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 经纪人约看客户后，需要知道录音中的改进点
- 传统方式依赖人工听录音反馈，效率低且不及时
- 需要标准化录音分析流程，给出可操作的改进建议

**录音分析的价值**：

| 价值 | 说明 |
|-----|------|
| 效率提升 | AI自动分析，1-3分钟内出结果 |
| 标准化 | 统一分析维度，避免遗漏 |
| 可操作 | 给出具体改进建议，不是泛泛而谈 |
| 辅助成交 | 通过话术改进提升约看转化率 |

### 1.2 目标

**做完之后期望达到什么效果？**

- 经纪人上传录音后，快速获得AI分析反馈
- 了解话术技巧改进方向
- 识别客户需求和意向
- 持续跟踪客户，提升约看转化率

## 8. 异常情况处理

### 8.1 异常类型

| 异常类型 | 处理方式 |
|---------|---------|
| 文件格式错误 | 提示"仅支持MP3/WAV格式"，引导重新选择 |
| 文件过大 | 提示"文件大小不能超过20MB"，引导压缩或重新录制 |
| 时长不符 | 提示"录音时长需在3-30分钟之间" |
| 网络中断 | 自动保存已上传部分，恢复后可继续上传 |
| AI分析失败 | 提示失败原因，引导重新分析或重新上传 |

### 8.2 异常处理原则

- 出现异常后给用户**合理的提示**，说明失败原因
- 引导用户**重新分析**或**重新上传音频文件**后分析
- 网络中断时**自动保存分析进度**（已上传部分），恢复后可继续

## 9. 关联管理

### 9.1 关联规则

| 规则 | 说明 |
|-----|------|
| 录音-楼盘 | 一段录音关联一个楼盘 |
| 录音-客户 | 一段录音至多关联**一位客户** |
| 解除关联 | 支持解除关联后重新关联 |

### 9.2 关联流程

```
录音上传页
    │
    ├── 点击"关联楼盘" → 楼盘选择页 → 选择后返回
    │
    └── 点击"关联客户" → 客户选择页 → 选择后返回
```

### 9.3 楼盘选择页

| 功能 | 说明 |
|-----|------|
| 页面标题 | 选择楼盘 |
| 搜索 | 关键词搜索楼盘名称 |
| 列表 | 单选列表，显示楼盘名称/区域/装修状态 |
| 确认 | 确认选择后返回上传页 |

### 9.4 客户选择页

| 功能 | 说明 |
|-----|------|

| 列表 | 单选列表，显示客户姓名/电话/意向楼盘 |
| 确认 | 确认选择后返回上传页 |

| 指标 | 目标值 |
|-----|-------|

### 11.2 分析pipeline阶段

| 阶段 | 预计耗时 | 说明 |
|-----|---------|------|
| 音频上传 | 依文件大小 | 支持断点续传 |

| AI 分析 | 30-90 秒 | 使用大语言模型进行话术分析 |
| 报告生成 | < 5 秒 | 结构化输出分析结果 |
