## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 经纪人约看客户后，需要知道录音中的改进点
- 传统方式依赖人工听录音反馈，效率低且不及时
- 需要标准化录音分析流程，给出可操作的改进建议

**录音分析的价值**：

| 价值 | 说明 |
|-----|------|
| 效率提升 | AI自动分析，1-3分钟内出结果 |
| 标准化 | 统一分析维度，避免遗漏 |
| 可操作 | 给出具体改进建议，不是泛泛而谈 |

### 1.2 目标

**做完之后期望达到什么效果？**

- 经纪人上传录音后，快速获得AI分析反馈
- 了解话术技巧、需求挖掘、改进建议
- 持续跟踪客户，提升约看转化率

## 5. 核心业务流程

### 5.1 录音分析流程

```
上传录音
    ↓
选择录音类型（邀约/带看/逼单）
    ↓
关联新盘（可自动带入）
    ↓
关联客户（可选）
    ↓
提交分析
    ↓
AI处理（上传→转写→分析→报告）
    ↓
查看分析结果
```

### 5.2 分析阶段

| 阶段 | 说明 | 预计耗时 |
|------|------|---------|
| 上传 | 录音文件上传到服务器 | 视文件大小 |
| 转写 | 语音转文字 | 1-2分钟 |
| 分析 | AI分析录音内容 | 1-2分钟 |
| 报告 | 生成分析报告 | <1分钟 |

## 6. 功能详情

### 6.1 录音上传

| 字段 | 说明 | 必填 |
|-----|------|------|
| 录音类型 | 邀约/带看/逼单 | ✅ |
| 关联新盘 | 选择关联的新盘 | ✅ |
| 关联客户 | 选择关联的客户 | ❌ |
| 录音文件 | 上传录音文件（mp3/m4a/wav） | ✅ |
| 备注 | 补充说明 | ❌ |

### 6.2 AI分析维度

| 维度 | 说明 |
|-----|------|
| 话术技巧 | 开场白、产品介绍、异议处理、逼单技巧 |
| 需求挖掘 | 客户需求识别、预算了解、决策链 |
| 改进建议 | 具体可操作的改进点 |

### 6.3 分析报告

| 内容 | 说明 |
|-----|------|
| 总评分 | 1-100分 |
| 各维度评分 | 话术技巧、需求挖掘等 |
| AI建议 | 具体的改进建议 |
| 优秀片段 | 录音中表现好的部分 |
