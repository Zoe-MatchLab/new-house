## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 团队需要明确的组织架构和权限控制
- 不同角色需要不同的功能权限
- 管理层需要管理团队成员

**当前存在什么问题？**

- 权限边界模糊
- 角色功能混乱
- 人员管理缺乏统一入口

**不做会怎样？**

- 权限控制不安全
- 功能使用混乱
- 团队管理效率低

### 1.2 目标

**做完之后期望达到什么效果？**

- 清晰的组织架构管理
- 明确的角色权限控制
- 便捷的成员管理功能

**如何衡量成功？**

| 指标 | 目标值 |
|-----|-------|
| 权限匹配率 | 100% |
| 组织同步 | 实时同步 |
| 管理效率 | 提升50% |

## 2. 用户故事

### 2.1 用户故事列表

| ID | 角色 | 用户故事 | 验收标准 | 优先级 |
|----|-----|---------|---------|-------|
| US-01 | 管理员 | 我想查看组织架构，以便了解团队结构 | 树状组织架构展示 | P0 |
| US-02 | 管理员 | 我想同步企业微信组织架构，以便保持人员最新 | 一键同步企业微信 | P0 |
| US-03 | 管理员 | 我想管理成员权限，以便控制功能访问 | 权限配置/修改 | P0 |
| US-04 | 区域经理 | 我想查看本团队成员，以便了解团队情况 | 成员列表、筛选 | P1 |
| US-05 | 成员 | 我想查看我的角色和权限，以便了解可使用的功能 | 角色说明、权限列表 | P1 |

### 2.2 详细用户故事

#### US-01：组织架构查看

**角色**：管理员
**场景**：当我需要了解团队结构时
**期望**：我可以查看完整的组织架构树

**验收标准**：
```
Given 管理员进入组织架构页面
When 查看组织结构
Then 显示树状结构：
  - 事业部 → 区域 → 大区 → 门店 → 经纪人
And 每个节点显示：名称、人数、状态
And 支持展开/收起、搜索
```

#### US-03：权限管理

**角色**：管理员
**场景**：当我需要配置权限时
**期望**：我可以为不同角色配置不同权限

**验收标准**：
```
Given 管理员进入权限配置
When 选择角色（如区域经理）
Then 显示该角色的功能权限
And 可开启/关闭特定权限
And 保存后实时生效
```

## 3. 功能概述

### 3.1 核心功能

```
团队管理
├── 组织架构管理
│   ├── 架构展示（树状）
│   ├── 组织同步（企业微信）
│   └── 成员管理（增删改查）
│
├── 权限管理
│   ├── 角色定义
│   ├── 权限配置
│   └── 权限验证
│
└── 角色说明
    ├── 管理员/运营
    ├── 区域经理
    ├── 店长
    └── 经纪人
```

### 3.2 角色权限矩阵

| 功能 | 管理员/运营 | 区域经理 | 店长 | 经纪人 |
|-----|-------------|----------|------|--------|
| 新盘录入 | ✅ | ❌ | ❌ | ❌ |
| 任务配置 | ✅ | ✅ | ❌ | ❌ |
| 任务下发 | ✅ | ✅ | ❌ | ❌ |
| 任务执行 | ✅ | ❌ | ❌ | ✅ |
| 话术训练 | ✅ | ❌ | ❌ | ✅ |
| 录音分析 | ✅ | ❌ | ❌ | ✅ |
| 数据查看 | ✅ | ✅ | ✅ | 仅本人 |

### 3.3 组织架构

```
公司
└── 事业部
    └── 区域
        └── 大区
            └── 门店
                └── 经纪人
```

**说明**：店长同时也是经纪人（双重角色）

| 权限验证 | 登录时验证 |

## 5. 验收标准

### 5.1 功能验收标准

| 功能点 | 验收条件（Given-When-Then） | 优先级 | 状态 |
|-------|---------------------------|-------|------|
| 架构展示 | Given 管理员进入 When 查看组织 Then 树状展示 | P0 | 🆕 |
| 组织同步 | Given 点击同步 When 同步企业微信 Then 更新组织架构 | P0 | 🆕 |
| 成员查看 | Given 选择组织节点 When 查看成员 Then 成员列表 | P0 | 🆕 |
| 权限配置 | Given 管理员配置 When 选择角色并修改 Then 保存生效 | P0 | 🆕 |
| 权限查看 | Given 用户查看 When 进入个人中心 Then 显示角色权限 | P1 | 🆕 |
| 角色切换 | Given 双重角色 When 切换身份 Then 显示对应功能 | P1 | 🆕 |

### 5.2 性能验收标准

| 指标 | 目标值 | 说明 |
|-----|-------|-----|
| 同步速度 | ≤30秒 | 同步企业微信 |
| 页面加载 | ≤2秒 | 组织架构加载 |
| 查询响应 | ≤1秒 | 成员搜索 |
