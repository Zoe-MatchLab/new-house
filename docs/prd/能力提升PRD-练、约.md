## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 经纪人需要快速掌握楼盘产品力和销售话术
- 传统培训依赖人工考核，效率低且反馈慢
- 经纪人需要知道约看录音中的改进点，持续优化

**当前存在什么问题？**

- 10分钟通关靠背书，人工考核占用大量时间
- 话术改进靠经验，不知道具体哪里要改
- 录音分析缺少标准化流程，反馈周期长

**不做会怎样？**

- 经纪人能力提升缓慢
- 约看转化率难以提升
- 团队整体战力无法量化

### 1.2 目标

**做完之后期望达到什么效果？**

- 经纪人可随时AI对话练习，快速掌握卖点
- AI实时分析录音，给出可操作的改进建议
- 能力评估量化可视化，管理层可针对性辅导

**如何衡量成功？**

| 指标 | 目标值 |
|-----|-------|
| 训练覆盖率 | ≥80%经纪人参与 |
| 能力提升率 | 较上一周期提升15% |
| 录音分析响应 | ≤3分钟 |
| 约看转化率 | 提升10% |

### 1.3 成功指标

| 指标 | 当前值 | 目标值 | 衡量方式 |
|-----|-------|-------|---------|
| 人均训练时长 | 无统计 | ≥30分钟/周 | 系统统计 |
| AI评分≥80分占比 | 无 | ≥70% | 系统统计 |
| 录音分析量 | 无 | ≥3次/周 | 系统统计 |
| 约看成功率 | 无统计 | 提升10% | 对比数据 |

### 2.3 详细用户故事

#### US-T01：AI话术训练

**角色**：经纪人
**场景**：当我想进行话术练习时
**期望**：我选择楼盘和场景，AI扮演客户进行对话

**验收标准**：
```
Given 经纪人进入训练页面
When 选择楼盘（龙湖·云景）和场景（产品介绍）
And 点击"开始训练"
Then AI扮演客户开始提问，经纪人扮演销售回答

Given 对话过程中
When 经纪人回答后
Then AI实时评分（1-10分）+ 1-2条改进建议

Given 每场景10-15轮对话后
When 点击"结束训练"
Then 显示综合评分（满分100）+ 能力雷达图 + 分维度改进建议
And 记录训练时长、对话轮数到数据库
```

#### US-R01：录音分析

**角色**：经纪人
**场景**：当我需要分析约看录音时
**期望**：上传后快速获得分析结果

**验收标准**：
```
Given 经纪人上传录音文件（MP3/WAV，≤20MB）
When 点击"开始分析"
Then 显示分阶段进度：
  - ⬆️ 音频上传中（显示进度百分比）
  - 📝 音频转写中（预计时间：录音时长30-50%）
  - 🤖 AI分析中（预计时间：30-90秒）
  - ✅ 分析完成

Given 95%录音
Then 分析完成时间 ≤3分钟

Given 分析完成
Then 自动展示结果：话术评分、优点、改进建议
And 无需用户刷新或等待推送
```

## 3. 功能概述

### 3.1 核心功能

```
能力提升
├── 话术训练
│   ├── PC管理后台
│   │   ├── 题库管理（按楼盘/场景配置）
│   │   ├── 训练数据查看
│   │   └── 能力看板
│   │
│   └── 小程序端
│       ├── 训练入口（新盘上线后显示）
│       ├── 场景选择
│       ├── AI对话练习
│       ├── 能力评估（综合评分+雷达图）
│       └── 训练记录
│
└── 约看录音分析
    ├── 小程序端
    │   ├── 录音上传（MP3/WAV，≤20MB）
    │   ├── 实时进度展示
    │   ├── AI分析结果（话术技巧/客户需求/改进建议）
    │   └── 客户关联
    │
    └── PC管理后台（可选）
        └── 录音管理/数据分析
```

### 3.2 场景类型

| 场景 | 说明 |
|-----|------|
| 产品介绍 | 楼盘卖点讲解 |
| 邀约客户 | 电话邀约技巧 |
| 首次带看 | 带看流程话术 |
| 逼定 | 促成签约技巧 |
| 异议处理 | 客户异议应对 |
| 竞品对比 | 竞品优劣势话术 |
| 其他 | 自定义场景 |

### 3.3 AI分析维度

| 维度 | 说明 |
|-----|------|
| 话术技巧 | 开场白、需求挖掘、异议处理、促成技巧 |
| 情绪判断 | 客户意向度（高/中/低） |
| 客户需求 | 客户关注点、潜在需求、购买意向 |
| 改进建议 | 具体可操作的提升建议 |

### 3.4 能力评估模型

| 维度 | 说明 |
|-----|------|
| 产品熟悉度 | 对楼盘信息的掌握程度 |
| 话术逻辑 | 沟通逻辑和表达条理性 |
| 应变能力 | 应对客户异议的反应 |
| 情绪把控 | 情绪管理和亲和力 |

### 3.5 能力等级

| 等级 | 说明 |
|-----|------|
| 单盘专家 | 单个楼盘能力达标 |
| 商圈专家 | 商圈内多楼盘能力达标 |
| 行政区专家 | 行政区内能力领先 |

### 3.6 关联模块

## 4. 业务规则

### 4.1 话术训练规则

| 规则 | 说明 |
|-----|------|
| 题库来源 | 人工创建（PC端） |
| 题目格式 | 问答对 |
| 对话轮数 | 每场景10-15轮 |
| 必答题 | 可配置每场景X道必答题 |
| 超时处理 | 可选择继续或结束 |
| 合格标准 | 综合评分≥80分，每个维度≥60分 |
| 未达标 | 可重新训练 |

### 4.2 约看录音规则

| 规则 | 说明 |
|-----|------|
| 录音来源 | 小程序录音入口上传 |
| 录音格式 | MP3/WAV |
| 文件大小 | ≤20MB |
| 录音时长 | 3-30分钟 |
| 分析响应 | 1-3分钟（95%录音），最长5分钟 |
| 客户关联 | 一段录音至多关联一位客户 |

### 4.3 AI反馈规则

| 阶段 | 反馈内容 |
|-----|---------|
| 每轮回答后 | 即时评分（1-10分）+ 1-2句话术建议 |
| 训练结束 | 综合评分 + 能力雷达图 + 分维度改进建议 |
| 录音分析 | 话术评分 + 优点 + 改进建议 |

### 4.4 异常处理

| 场景 | 处理方式 |
|-----|---------|
| 网络中断（录音） | 自动保存进度，恢复后可继续 |
| 分析失败 | 提示原因，引导重新分析 |
| 录音格式错误 | 提示支持的格式 |
| 超时未完成 | 提示预计剩余时间 |
