## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 探盘任务产出的物料需要统一管理
- 经纪人需要便捷获取推广素材
- 管理层需要审核和筛选优质内容

**当前存在什么问题？**

- 物料分散在各处，无统一管理
- 经纪人获取素材需要找各种渠道
- 优质内容无法快速识别和复用

**不做会怎样？**

- 物料重复利用率低
- 经纪人推广效率受影响
- 内容质量无法把控

### 1.2 目标

**做完之后期望达到什么效果？**

- 统一物料管理，按楼盘/类型/任务分类
- 经纪人可快速获取和分享素材
- 管理层可审核优质内容

**如何衡量成功？**

| 指标 | 目标值 |
|-----|-------|
| 物料覆盖率 | 100%上线楼盘 |
| 下载/分享量 | ≥5次/楼盘/月 |
| 分享成功率 | ≥95% |

### 1.3 成功指标

| 指标 | 当前值 | 目标值 | 衡量方式 |
|-----|-------|-------|---------|
| 物料数量 | 无统计 | ≥100个/楼盘 | 系统统计 |
| 分享次数 | 无 | ≥5次/周 | 系统统计 |
| 审核效率 | 无 | ≤24小时 | 系统统计 |

## 2. 用户故事

### 2.1 用户故事列表

| ID | 角色 | 用户故事 | 验收标准 | 优先级 |
|----|-----|---------|---------|-------|
| US-01 | 经纪人 | 我想查看楼盘的推广物料，以便进行获客 | 按楼盘查看所有物料 | P0 |
| US-02 | 经纪人 | 我想下载或分享物料，以便推广使用 | 支持下载/一键分享 | P0 |
| US-03 | 经纪人 | 我想生成物料海报，以便朋友圈分享 | 带小程序码的海报图 | P0 |
| US-04 | 运营人员 | 我想管理物料库，以便审核和筛选 | 物料审核/分类管理 | P1 |
| US-05 | 区域经理 | 我想查看物料使用情况，以便了解推广效果 | 物料使用统计 | P1 |

### 2.2 详细用户故事

#### US-01：查看物料

**角色**：经纪人
**场景**：当我需要获取楼盘推广素材时
**期望**：我可以按楼盘查看所有相关物料

**验收标准**：
```
Given 经纪人进入物料库
When 选择楼盘（龙湖·云景）
Then 显示该楼盘所有物料：
  - 探盘原图（照片、视频）
  - AI生成文案（小红书、公众号）
  - AI生成图片（封面、配图）
  - AI生成视频素材（脚本、口播文字）

Given 物料按类型分类
When 选择分类筛选
Then 显示对应类型的物料
```

#### US-03：生成物料海报

**角色**：经纪人
**场景**：当我需要分享到朋友圈时
**期望**：我可以生成带有小程序码的物料海报

**验收标准**：
```
Given 经纪人选择物料
When 点击"生成海报"
Then 系统生成带小程序码的海报图
And 可保存到本地或直接分享到朋友圈
And 海报包含：物料预览 + 小程序码 + 来源标识
```

## 3. 功能概述

### 3.1 核心功能

```
物料管理
├── 物料类型
│   ├── 探盘原图（照片、视频）
│   ├── AI生成文案（小红书、公众号）
│   ├── AI生成图片（封面、配图）
│   └── AI生成视频素材（脚本、口播文字）
│
├── PC管理后台
│   ├── 物料库（按楼盘查看）
│   ├── 素材分类（类型/任务/楼盘/渠道）
│   └── 权限配置
│
└── 小程序端
    ├── 物料查看/预览/下载
    └── 一键分享（微信/朋友圈/群）
```

### 3.2 物料分类维度

| 维度 | 分类 |
|-----|------|
| 按内容类型 | 文本、图片、视频、音频 |
| 按任务 | 探盘路书、话术训练、约看录音 |
| 按楼盘 | 各上线楼盘 |
| 按渠道 | 小红书、公众号、视频号、朋友圈 |

### 3.3 分享功能

| 方式 | 说明 |
|-----|------|
| 微信分享 | 分享给好友/群聊 |
| 朋友圈 | 生成海报图片保存后发朋友圈 |
| 下载 | 保存到本地 |

### 3.4 关联模块

| 关联模块 | 关联方式 |
|---------|---------|
| 探盘任务 | 任务产出的物料自动进入物料库 |
| 新盘管理 | 上线楼盘自动生成物料目录 |
