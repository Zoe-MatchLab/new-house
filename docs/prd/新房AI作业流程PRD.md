## 1. 背景与目标

### 1.1 需求背景

随着新房市场竞争加剧，经纪人获取楼盘信息、了解产品卖点、掌握销售话术的能力成为成交的关键。然而，传统模式下：

- **信息获取滞后**：经纪人依赖线下踩盘或转发资料，无法快速全面了解楼盘
- **能力参差不齐**：产品知识掌握程度不一，导致约看率、成交率不稳定
- **物料质量不一**：探盘成果标准不统一，生成的新媒体内容质量难以保障
- **管理抓手缺失**：管理层难以实时掌握经纪人任务执行进度，缺乏数据支撑

### 1.2 问题分析

| 问题 | 影响 | 解决方案 |
|-----|------|---------|
| 信息获取滞后 | 经纪人无法快速了解楼盘 | 标准化新盘录入，81字段结构化存储 |
| 能力参差不齐 | 约看率、成交率不稳定 | AI话术训练，实时反馈 |
| 物料质量不一 | 内容无法保障 | 标准化探盘任务，AI辅助生成 |
| 管理抓手缺失 | 缺乏数据支撑 | 数据看板，实时追踪 |

### 1.3 目标

**做完之后期望达到什么效果？**

- 经纪人可以快速了解楼盘，标准化产出物料
- AI辅助话术训练，快速提升专业能力
- 新媒体获客内容自动生成，降低成本
- 管理层有数据看板，实时掌握团队状态

**如何衡量成功？**

| 指标 | 目标值 |
|-----|-------|
| 新盘录入完成率 | ≥95%字段完整度 |
| 探盘任务完成率 | ≥90%（截止前完成） |
| AI话术训练覆盖率 | ≥80%经纪人参与 |
| 内容发布量 | 月均≥5条/经纪人 |
| 约看转化率 | 提升15% |

### 1.4 成功指标

| 指标 | 当前值 | 目标值 | 衡量方式 |
|-----|-------|-------|---------|
| 经纪人平均学习时长 | 无标准 | ≥30分钟/周 | 系统统计 |
| 内容生产效率 | 人工2小时/篇 | AI生成≤5分钟/篇 | 操作计时 |
| 任务执行完成率 | 无统计 | ≥90% | 系统统计 |
| 约看录音分析量 | 无 | ≥3次/周 | 系统统计 |

### 2.2 详细用户故事

#### US-01：录入新盘基础信息

**角色**：新房项目维护人员
**场景**：当我要添加一个新楼盘时
**期望**：我希望快速录入完整信息，系统帮我校验必填项

**验收标准**：
```
Given 新房维护人员进入"新建楼盘"页面
When 填写楼盘名称、地址、均价等必填字段
And 点击"保存"
Then 系统保存成功，页面提示"保存成功"
And 楼盘状态变为"草稿"

Given 新房维护人员填写信息时遗漏必填字段
When 点击"保存"
Then 系统提示"请填写完整必填信息"
And 高亮缺失字段

Given 新房维护人员上传多张图片
When 选择图片文件（≤10MB/张，最多20张）
Then 图片显示上传进度，完成后分类展示
```

#### US-04：下发探盘任务

**角色**：区域经理
**场景**：当新盘上线后需要下发探盘任务时
**期望**：我希望配置执行人和截止时间，一键下发

**验收标准**：
```
Given 区域经理在楼盘详情页点击"下发任务"
When 配置执行人（组织架构或指定人员）+ 截止时间
And 点击"确认下发"
Then 任务状态变为"进行中"
And 执行人收到站内通知+小程序推送

Given 任务进行中
When 截止时间前经纪人未完成
Then 系统发送提醒通知（轻度→中度→严重）
```

#### US-05：执行探盘任务

**角色**：经纪人
**场景**：当我收到探盘任务需要执行时
**期望**：我希望按流程完成打卡、采集、AI内容生成

**验收标准**：
```
Given 经纪人到达楼盘现场
When 点击"定位打卡"
Then 系统校验位置（200米内通过，200-500米警告，500米以上拦截）

Given 定位打卡成功
When 按配置完成各点位资料采集（拍照/视频/文字）
Then AI自动初审（清晰度、时长等）
And 采集完成进入下一阶段

Given 点位采集完成
When 选择需要生成的内容类型
Then AI生成对应内容（小红书文案/公众号图文/短视频脚本）
And 经纪人确认提交后任务完成
```

#### US-06：AI话术训练

**角色**：经纪人
**场景**：当我需要进行话术练习时
**期望**：我希望选择场景，与AI对话练习，获得实时反馈

**验收标准**：
```
Given 经纪人选择楼盘和训练场景
When 点击"开始训练"
Then AI扮演客户，经纪人扮演销售，开始对话

Given 对话过程中
When 经纪人回答问题
Then AI实时评价回答，给出改进建议

Given 对话完成
When 点击"结束训练"
Then 系统给出综合评分和能力维度分析
And 记录训练时长、评分到数据库
```

## 3. 功能概述

### 3.1 核心功能

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         新房AI作业流程                                   │
│                                                                         │
│    新盘管理 ──→ 探盘任务 ──→ 话术训练 ──→ 物料管理 ──→ 数据看板         │
│        │              │              │              │              │      │
│        ▼              ▼              ▼              ▼              ▼      │
│    81字段录入      定位打卡        AI对话         内容审核         发布数据  │
│    图片资料        点位采集        能力评估        AI生成         获客转化  │
│    审核上下线      AI内容生成      题库管理        物料库          能力评估  │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 六大模块

| 模块 | PC端功能 | 小程序端功能 |
|-----|---------|------------|
| **新盘管理** | 录入、审核、上线配置 | 查看楼盘信息 |
| **探盘任务** | 任务下发、进度追踪 | 执行任务（打卡/采集/提交） |
| **话术训练** | 题库配置、数据查看 | AI对话练习、能力评估 |
| **约看录音** | 录音管理、数据分析 | 录音上传、分析查看 |
| **物料管理** | 物料库、AI内容审核 | 查看物料 |
| **数据看板** | 全部数据、团队数据 | 个人数据、团队排名 |

### 3.3 功能流程

```
【第一阶段：新盘上线】
录入信息 → 保存草稿 → 提交审核 → 审核通过 → 上线配置 → 上线
                               ↓
                            驳回 → 修改 → 重新提交

【第二阶段：任务并行执行】
         ┌──────────────────┬──────────────────┐
         ▼                  ▼                  ▼
    探盘路书            话术训练            约看录音
    (必选)              (可选)              (可选)
    └─定位打卡          └─AI对话练习        └─录音上传
    └─点位采集          └─能力评估          └─AI分析
    └─AI内容生成

【第三阶段：数据沉淀】
任务执行 → 物料库 → 内容发布 → 获客转化 → 数据看板
```

### 3.4 关联模块

| 关联模块 | 关联方式 |
|---------|---------|
| 新盘管理 | 是源头，上线后触发探盘任务配置 |
| 探盘任务 | 依赖新盘信息，产出物料进入物料库 |
| 话术训练 | 依赖新盘上线配置，可关联多个题库 |
| 约看录音 | 可选功能，关联客户管理 |
| 物料管理 | 汇聚探盘成果和AI生成内容 |
| 数据看板 | 汇聚所有模块数据，进行统计分析 |

## 4. 业务规则

### 4.1 新盘管理规则

| 规则类型 | 规则内容 |
|---------|---------|
| **录入规则** | 81字段区分必填/非必填，支持批量导入Excel |
| **审核规则** | 当前人工审核，驳回需填写原因 |
| **上线规则** | 上线后自动通知可下发人员、关联训练题库 |
| **下线规则** | 下线前检查进行中任务，通知相关人员 |

### 4.2 探盘任务规则

| 规则类型 | 规则内容 |
|---------|---------|
| **任务配置** | 配置执行人、可见范围、截止时间、提醒节点 |
| **并行执行** | 探盘路书、话术训练、约看录音三者独立并行 |
| **完成判定** | 探盘路书：文案确认即完成；整体任务：全部子任务完成 |
| **打卡校验** | 200米内通过，200-500米警告，500米以上拦截 |

### 4.3 话术训练规则

| 规则类型 | 规则内容 |
|---------|---------|
| **题库管理** | 人工创建题库，按场景分类（产品介绍/邀约客户/异议处理等） |
| **训练入口** | 新盘上线后小程序首页显示训练入口 |
| **能力评估** | 综合评分 + 能力维度分析（产品力/话术/应变等） |

### 4.4 约看录音规则

| 规则类型 | 规则内容 |
|---------|---------|
| **录音格式** | MP3/WAV，最大50MB |
| **AI分析** | 话术技巧、客户需求、改进建议 |
| **客户关联** | 可选择关联现有客户或新建客户 |

### 4.5 特殊场景处理

| 场景 | 处理方式 |
|-----|---------|
| 必填字段缺失 | 提示"请填写完整必填信息"，高亮缺失字段 |
| 图片上传失败 | 显示"上传失败"，提供重试按钮 |
| 定位打卡偏移 | 200米内通过，200-500米警告，500米以上拦截 |
| 任务逾期 | 状态变更为"已失效"，不可继续完成 |
| 网络中断 | 自动保存草稿，恢复后可继续编辑 |
| 重复楼盘 | 提示"该楼盘名称已存在"，是否继续创建 |
