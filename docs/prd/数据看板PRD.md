## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 管理层需要实时掌握业务数据
- 各系统数据分散，无法形成全局视角
- 经纪人需要了解个人业绩和成长情况

**当前存在什么问题？**

- 各系统数据分散，看不到全貌
- 经纪人不知道自己的排名和差距
- 无法量化评估经纪人能力

**不做会怎样？**

- 管理层决策缺少数据支撑
- 经纪人缺乏目标和动力
- 无法评估培训效果

### 1.2 目标

**做完之后期望达到什么效果？**

- 数据实时汇总，可视化展示
- 按角色分层，查看权限范围内的数据
- 支持导出，便于汇报分析

**如何衡量成功？**

| 指标 | 目标值 |
|-----|-------|
| 数据更新 | 手动刷新，实时 |
| 覆盖率 | 100%功能模块 |
| 导出成功率 | ≥99% |

### 1.3 成功指标

| 指标 | 当前值 | 目标值 | 衡量方式 |
|-----|-------|-------|---------|
| 数据完整性 | 分散 | 全量数据 | 系统统计 |
| 查询响应 | 无 | ≤2秒 | 系统统计 |
| 使用率 | 无 | ≥80%用户 | 日活统计 |

### 2.2 详细用户故事

#### US-01：个人业绩看板

**角色**：经纪人
**场景**：当我需要了解个人业绩时
**期望**：我可以看到本月发布量、咨询量、报备量等数据

**验收标准**：
```
Given 经纪人进入个人看板
When 查看当前周期数据
Then 显示：
  - 发布量（小红书/公众号/视频号/朋友圈）
  - 咨询量（客户主动咨询次数）
  - 线索量（留资客户数量）
  - 报备量（报备楼盘数量）
  - 带看量（带看次数）
```

#### US-04：团队数据看板

**角色**：区域经理
**场景**：当我需要管理团队时
**期望**：我可以查看团队整体数据和排名

**验收标准**：
```
Given 区域经理进入团队看板
When 选择团队和时间范围
Then 显示：
  - 团队整体数据（发布量/咨询量/线索量/报备量/带看量）
  - 同比环比（较上一周期变化）
  - 业绩排名（团队内经纪人排名）
  - 任务跟进（进行中/逾期/已完成）
```

## 3. 功能概述

### 3.1 核心功能

```
数据看板
├── 内容发布指标
│   ├── 发布量（各平台）
│   ├── 浏览量（曝光）
│   └── 投流费用（推广消耗）
│
├── 获客转化指标
│   ├── 咨询量
│   ├── 线索量
│   ├── 报备量
│   ├── 带看量
│   └── 获客成本
│
├── 产品力训练指标
│   ├── 专家占比
│   ├── 平均能力分
│   └── 能力提升率
│
└── 数据导出
    ├── Excel导出
    └── PDF导出
```

### 3.2 指标说明

**内容发布指标**：

| 指标 | 说明 |
|-----|------|
| 发布量 | 各平台（小红书、公众号、视频号、朋友圈）发布数量 |
| 浏览量 | 各平台内容曝光量 |
| 投流费用 | 付费推广消耗 |

**获客转化指标**：

| 指标 | 说明 |
|-----|------|
| 咨询量 | 客户主动咨询总次数 |
| 线索量 | 留资客户数量 |
| 报备量 | 客户报备楼盘数量 |
| 带看量 | 经纪人带看次数 |
| 获客成本 | 单个有效客户的获取成本 |

**产品力训练指标**：

| 指标 | 说明 |
|-----|------|
| 专家占比 | 各级别专家人数占比 |
| 平均能力分 | 团队整体能力均值 |
| 能力提升率 | 较上一周期能力提升幅度 |

### 3.3 角色分层

| 角色 | 数据范围 |
|-----|---------|
| 经纪人 | 个人数据 |
| 区域经理 | 团队数据 |
| 管理层 | 全量数据 |

### 3.4 关联模块

| 关联模块 | 关联方式 |
|---------|---------|
| 新盘管理 | 楼盘基础数据 |
| 探盘任务 | 任务执行数据 |
| 能力提升 | 训练数据、能力评分 |
| 物料管理 | 内容发布数据 |
