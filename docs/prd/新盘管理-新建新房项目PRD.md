## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 当前经纪人依赖线下踩盘或转发资料，无法快速全面了解楼盘
- 新盘信息分散在各个渠道，管理人员无法统一管理
- 需要一个标准化的入口，将新盘信息结构化录入系统

**当前存在什么问题？**

- 新盘录入没有统一标准，信息散乱不完整
- 录入后无法快速关联到探盘任务、话术训练等后续流程
- 缺少审核机制，信息质量无法保证

**不做会怎样？**

- 后续探盘任务、话术训练等模块无法正常流转
- 数据看板缺少楼盘基础数据支撑

### 1.2 目标

**做完之后期望达到什么效果？**

- 管理人员可以通过PC后台快速录入新盘信息
- 新盘信息结构化存储，支持后续流程关联
- 录入信息有审核机制，保证数据质量

**如何衡量成功？**

| 指标 | 目标值 |
|-----|-------|
| 新盘录入完成率 | ≥95%字段完整度 |
| 审核通过率 | ≥90% |
| 录入效率 | 单个楼盘≤15分钟 |

### 1.3 成功指标

| 指标 | 当前值 | 目标值 | 衡量方式 |
|-----|-------|-------|---------|
| 录入时长 | 无标准 | ≤15分钟/楼盘 | 操作计时 |
| 字段完整度 | 参差不齐 | ≥95% | 系统统计 |
| 审核效率 | 人工判断 | ≤5分钟/楼盘 | 系统统计 |

## 3. 功能概述

### 3.1 核心功能

1. **新盘信息录入**
   - 基础信息录入（楼盘名称、地址、开发商等）
   - 价格信息录入（均价、总价区间、首付等）
   - 配套信息录入（交通、商业、教育、医疗等）
   - 楼栋信息录入（楼栋数、户数、户型等）

2. **图片资料上传**
   - 支持多张图片上传（户型图、沙盘图、样板间图、区位图等）
   - 图片分类展示
   - 图片预览和删除

3. **信息审核流程**
   - 提交审核 → 待审核 → 审核通过/驳回
   - 驳回时填写原因
   - 修改后可重新提交

4. **楼盘上下线**
   - 上线操作触发后续流程（任务配置、通知等）
   - 下线操作需确认无进行中任务

### 3.2 功能流程

```
录入信息 → 保存草稿 → 提交审核 → 审核通过 → 上线
                              ↓
                           驳回 → 修改 → 重新提交
```

### 3.3 关联模块

| 关联模块 | 关联方式 |
|---------|---------|
| 探盘任务 | 上线后自动关联，可配置探盘任务 |
| 话术训练 | 上线后自动关联训练题库 |
| 物料管理 | 上线后楼盘资料进入物料库 |
| 数据看板 | 上线后楼盘数据进入统计分析 |

## 4. 业务规则

### 4.1 通用规则

| 规则 | 说明 |
|-----|-----|
| 录入权限 | 仅新房项目维护人员可录入 |
| 审核权限 | 仅审核人员可审核 |
| 上线权限 | 审核通过后可上线 |
| 编辑限制 | 待审核/已上线状态不可编辑 |

### 4.2 特殊场景/异常处理

| 场景 | 处理方式 |
|-----|---------|
| 必填字段缺失 | 提示"请填写完整必填信息"，高亮缺失字段 |
| 图片上传失败 | 显示"上传失败"，提供重试按钮 |
| 重复楼盘 | 提示"该楼盘名称已存在"，是否继续创建 |
| 审核超时 | 超过48小时未审核，自动提醒审核人员 |
| 网络中断 | 自动保存草稿，恢复后可继续编辑 |
