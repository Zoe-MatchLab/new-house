## 1. 背景与目标

### 1.1 需求背景

**为什么要做这个需求？**

- 经纪人需要到楼盘现场采集真实物料（照片、视频、文字）
- 传统方式缺乏标准化流程，物料质量参差不齐
- 需要AI辅助生成新媒体内容，降低经纪人创作门槛

**当前存在什么问题？**

- 经纪人自行踩盘，信息记录不统一
- 物料采集后无人审核，质量无法保证
- 内容创作依赖经纪人个人能力，效果不稳定

**不做会怎样？**

- 后续物料管理缺少数据源
- AI生成内容缺少真实素材支撑
- 经纪人能力提升缺少实践场景

### 1.2 目标

**做完之后期望达到什么效果？**

- 标准化探盘流程，确保物料质量统一
- AI辅助内容生成，降低经纪人创作成本
- 任务进度可视化，管理层实时掌握执行情况

**如何衡量成功？**

| 指标 | 目标值 |
|-----|-------|
| 任务完成率 | ≥90%（截止前完成） |
| 物料合格率 | ≥95% |
| 内容生成效率 | AI生成≤30秒/篇 |
| 打卡位置合规率 | ≥98% |

### 1.3 成功指标

| 指标 | 当前值 | 目标值 | 衡量方式 |
|-----|-------|-------|---------|
| 任务完成率 | 无统计 | ≥90% | 系统统计 |
| 打卡合规率 | 无 | ≥98% | 系统记录 |
| 物料完整度 | 参差不齐 | ≥95% | 审核通过率 |
| 内容产出量 | 无 | ≥3条/任务 | 系统统计 |

### 2.2 详细用户故事

#### US-01：下发探盘任务

**角色**：区域经理
**场景**：当新盘上线后需要下发探盘任务时
**期望**：我希望配置执行人和截止时间，一键下发

**验收标准**：
```
Given 区域经理在楼盘详情页点击"下发任务"
When 配置执行人（组织架构或指定人员）+ 可见范围 + 截止时间 + 提醒节点
And 点击"确认下发"
Then 任务状态变为"待下发"自动变为"进行中"
And 执行人收到站内通知+小程序推送

Given 任务进行中
When 经纪人未在截止前完成
Then 系统按提醒节点发送提醒（轻度→中度→严重）
And 逾期后任务状态变为"已失效"
```

#### US-04：定位打卡

**角色**：经纪人
**场景**：当我到达楼盘现场需要打卡时
**期望**：我希望系统校验我的位置，确保真实性

**验收标准**：
```
Given 经纪人到达楼盘现场
When 点击"定位打卡"
Then 系统获取当前位置并校验：
  - 200米以内：✅ 正常通过
  - 200-500米：⚠️ 弹出警告，用户确认后可继续，记录"疑似偏移"
  - 500米以上：🚫 强制拦截，提示"打卡位置偏离过远"

Given 打卡偏移被拦截
When 用户申诉
Then 店长/管理员审批后可修正
And 记录"打卡偏移异常"
```

#### US-06：AI内容生成

**角色**：经纪人
**场景**：当我需要生成推广内容时
**期望**：我希望选择内容类型，AI自动生成

**验收标准**：
```
Given 点位采集完成
When 选择内容类型（小红书文案/公众号图文/短视频脚本）
Then AI在30秒内生成对应内容

Given 内容生成完成
When 经纪人确认提交
Then 探盘路书子任务完成
And 图片/视频后台继续异步生成

Given 需要重新生成
When 文案确认前
Then 可重新生成
And 文案确认后不可编辑物料，只能重做整个任务
```

## 3. 功能概述

### 3.1 核心功能

```
探盘任务
├── PC管理后台
│   ├── 任务配置（执行人、可见范围、截止时间）
│   ├── 任务下发/暂停/催办/编辑/删除
│   └── 进度追踪
│
└── 小程序端
    ├── 任务接收（通知）
    ├── 探盘路书（必选）
    │   ├── 定位打卡
    │   ├── 点位采集（拍照/视频/文字）
    │   └── AI内容生成
    ├── 话术训练（可选，并行）
    └── 约看录音（可选，并行）
```

### 3.2 任务结构

**三个子任务（并行执行，互不依赖）**：

| 子任务 | 必选 | 说明 |
|-------|-----|------|
| 探盘路书 | ✅ | 生成新媒体内容 |
| 话术训练 | ❌ | AI话术练习（见能力提升模块） |
| 约看录音 | ❌ | 录音分析（见能力提升模块） |

### 3.3 功能流程

```
【任务下发】
新盘上线 → 配置任务 → 下发任务 → 通知执行人

【任务执行】（小程序端）
┌─────────────────────────────────────────────┐
│  阶段1：定位打卡                              │
│  ├── 到达楼盘现场                            │
│  ├── 系统定位校验（范围内才可打卡）               │
│  └── 确认到达                                │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  阶段2：点位采集                              │
│  ├── 按配置完成各点位资料采集                   │
│  ├── 采集方式：拍照/视频/录音/文字              │
│  └── AI初审：照片清晰度、视频时长等自动检测       │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  阶段3：文案确认                              │
│  ├── 经纪人选择需要生成的内容类型                │
│  ├── AI生成对应内容                           │
│  └── 经纪人确认提交                           │
│      └── ⭐ 确认后探盘路书子任务完成            │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  阶段4：AI内容生成（后台异步执行）              │
│  ├── 生成各平台内容（可退出小程序）              │
│  ├── 实时进度展示                             │
│  ├── 生成完成后推送通知                        │
│  └── 经纪人确认成果                           │
└─────────────────────────────────────────────┘
```

### 3.4 任务完成判定

| 判定 | 说明 |
|-----|------|
| 探盘路书子任务 | 文案确认后即完成（图片/视频为耗时任务，后台继续生成） |
| 整个探盘任务 | 探盘路书 + 话术训练 + 约看录音**全部完成**才算完成 |
| 无需审核 | 提交后直接进入下一状态 |

### 3.5 关联模块

| 关联模块 | 关联方式 |
|---------|---------|
| 新盘管理 | 依赖楼盘信息，入口从楼盘详情页 |
| 能力提升 | 话术训练、约看录音可选并行 |
| 物料管理 | 探盘成果进入物料库 |

## 4. 业务规则

### 4.1 任务配置规则

| 规则 | 说明 |
|-----|------|
| 新建入口 | 从楼盘详情页 → 上线配置 → 发起探盘任务（不在任务列表页新建） |
| 执行人配置 | 按组织架构（事业部→区域→大区→门店）或指定人员 |
| 可见范围 | 选择部门后，该部门及所有下级部门、人员自动纳入 |
| 任务项配置 | 打卡/路书/训练/录音均支持可选或必填配置 |
| 下发规则 | 新盘上线时配置的可下发人员可下发任务 |

### 4.2 打卡规则

| 偏移范围 | 处理方式 |
|---------|---------|
| 200米以内 | ✅ 正常打卡通过 |
| 200-500米 | ⚠️ 弹出警告，用户确认后可继续，记录"疑似偏移" |
| 500米以上 | 🚫 强制拦截，记录"打卡偏移异常" |

**申诉机制**：用户可申诉 → 店长/管理员审批 → 24小时内处理

### 4.3 提醒规则

| 提醒级别 | 触发时间 | 提醒方式 |
|---------|---------|---------|
| 轻度提醒 | 截止前24小时 | 推送提醒通知 |
| 中度提醒 | 截止前12小时 | 强提醒 + 短信通知 |
| 严重提醒 | 截止前2小时 | 强提醒 + 电话催办 |

**逾期规则**：
- 逾期后**不可完成任务**
- 任务直接变更为"已失效"

### 4.4 内容生成规则

| 内容类型 | 说明 | 产出物 |
|---------|------|--------|
| 小红书文案 | 针对小红书平台风格 | 500-800字图文脚本 |
| 小红书封面图 | 吸引眼球的封面设计 | 3:4竖图 |
| 公众号文案 | 专业深度的楼盘介绍 | 1000-1500字图文脚本 |
| 公众号图片 | 配合文案的配图 | 多张横图 |
| 短视频脚本 | 15-60秒口播脚本 | 分镜头脚本 |
| 短视频内容 | AI生成口播文字+封面 | 可直接录制 |

### 4.5 异常处理

| 场景 | 处理方式 |
|-----|---------|
| 定位偏移200-500米 | 警告确认，记录疑似偏移 |
| 定位偏移500米以上 | 强制拦截，需申诉 |
| 任务逾期 | 状态变为"已失效"，不可继续 |
| 网络中断 | 自动保存进度，恢复后继续 |
| 内容生成失败 | 提示失败原因，可重新生成 |

## 5. 验收标准

### 5.1 功能验收标准

| 功能点 | 验收条件（Given-When-Then） | 优先级 | 状态 |
|-------|---------------------------|-------|------|
| 任务配置 | Given 区域经理进入任务配置 When 配置执行人和截止时间 Then 可下发任务 | P0 | 🆕 |
| 任务下发 | Given 配置完成 When 点击下发 Then 执行人收到通知，状态变为进行中 | P0 | 🆕 |
| 定位打卡 | Given 经纪人到达现场 When 点击打卡 Then 系统校验位置，200米内通过 | P0 | 🆕 |
| 打卡警告 | Given 位置偏移200-500米 When 点击打卡 Then 弹出警告，确认后可继续 | P0 | 🆕 |
| 打卡拦截 | Given 位置偏移500米以上 When 点击打卡 Then 强制拦截，提示偏移过远 | P0 | 🆕 |
| 点位采集 | Given 打卡成功 When 完成各点位采集 Then 进入下一阶段 | P0 | 🆕 |
| AI内容生成 | Given 点位采集完成 When 选择内容类型 Then AI生成对应内容≤30秒 | P0 | 🆕 |
| 文案确认 | Given AI生成内容 When 经纪人确认 Then 探盘路书任务完成 | P0 | 🆕 |
| 任务暂停 | Given 任务进行中 When 区域经理点击暂停 Then 状态变为待下发 | P0 | 🆕 |
| 任务催办 | Given 任务进行中 When 区域经理点击催办 Then 执行人收到提醒通知 | P0 | 🆕 |
| 任务查看 | Given 任何人查看任务 When 进入任务详情 Then 显示进度和状态 | P0 | 🆕 |
| 申诉处理 | Given 打卡异常 When 店长审批 Then 可修正打卡记录 | P1 | 🆕 |
| 任务重做 | Given 任务完成但物料效果不佳 When 经纪人申请 Then 允许重做 | P1 | 🆕 |

### 5.2 性能验收标准

| 指标 | 目标值 | 说明 |
|-----|-------|-----|
| 页面加载时间 | ≤2秒 | 任务列表、任务详情 |

| 内容生成速度 | ≤30秒 | 文案类内容 |

| 测试类型 | 测试关注点 |
